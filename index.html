Project Overview & Requirements

This software solution manages metadata for various media types, providing a persistent backend API and a user-friendly GUI.

System Architecture

Backend (Flask)

The backend handles data persistence (JSON/Pickle) and HTTP API requests.

It provides CRUD operations for all media types and communicates with the frontend via JSON.

Frontend (PyQt/PySide)

The frontend is a desktop GUI application.

It interacts with the backend API to display media, create/edit/delete entries, and perform searches.

Backend API Endpoints

List All Media: GET request to retrieve all media entries.

List by Category: GET request filtered by media category.

Display Metadata: GET request to fetch metadata of a specific item.

Search by Name: GET request with exact name match.

Create New Media: POST request to add a new entry.

Update Metadata: PUT request to edit an existing entry.

Delete Media: DELETE request to remove an entry.

GUI Features

Data Visualization: Shows lists of media with title, author, publication date, and category.

CRUD Forms: Forms for creating, editing, and deleting media entries interact with the backend API.

Interactive Search: Quickly filter or search the media library.

Interactive Demo

Client View: Displays the media list from the frontend interface. Click items to simulate API calls.

Backend Response: Shows simulated metadata responses from the backend for selected media items.

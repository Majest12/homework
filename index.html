Project Overview & Requirements

This project is a two-part software system, a Flask Backend and a PyQt/PySide Frontend built to manage metadata for an online library of books, films, and magazines. The Backend stores media information in JSON and provides RESTful APIs for listing, searching, filtering, creating, deleting, and viewing media entries. The Frontend uses these APIs to display the library, show detailed metadata, and handle user interactions. Users can add new media by clicking "Create New Media" and filling in the required details, or delete an entry by selecting it, clicking "Delete Selected Media," and confirming the action.

Media Categories

  • Book
  • Film
  • Magazine

Stored Metadata

  1. Name (Title)
  2. Publication date
  3. Author (or Director/Creator)
  4. Category

Backend (Flask)

The backend handles data persistence (JSON/Pickle) and HTTP API requests. It provides CRUD operations for all media types and exposes endpoints such as:

  • GET /media list all media
  • GET /media?category=Book filter by category
  • GET /media/{id} get metadata
  • POST /media create
  • PUT /media/{id} update
  • DELETE /media/{id} delete

Frontend (PyQt/PySide)

The frontend is a desktop GUI application that interacts with the backend API to display media lists, perform searches, and present metadata. It also offers create/edit/delete functionality via forms that call the API.

Backend (Flask) HTTP Endpoints

List Endpoints

List All Media: GET /media returns an array of media metadata objects.

List by Category: GET /media?category=<category> returns filtered results.

Display Metadata: GET /media/<id> returns the metadata for a single item.

Search & CRUD

Search by Name (Exact): GET /media?name=<name>

Create New Media: POST /media payload includes name, author, date, category.

Update Metadata: PUT /media/<id>

Delete Media: DELETE /media/<id>

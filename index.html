Project Overview & Requirements

This software solution manages metadata for various media types, providing a persistent backend API and a user-friendly graphical interface (GUI).

Media Categories

  • Book
  • Film
  • Magazine

Stored Metadata

  1. Name (Title)
  2. Publication date
  3. Author (or Director/Creator)
  4. Category

System Architecture

The system uses a two-tier architecture:

The Backend (Flask) handles data persistence (JSON/Pickle) and API requests. The Frontend (PyQt/PySide) is a desktop GUI application that acts as the client, communicating with the backend via HTTP/JSON.

Concept Documentation (Task 1)

Backend (Flask) HTTP Endpoints

The API provides the following endpoints for media management (CRUD operations):

# Function Method Endpoint (Example)
1 List All Media GET /media
2 List by Category GET /media/category/Book
3 Search by Name (Exact) GET /media/search?name=Dune
4 Display Metadata GET /media/123
5 Create New Media POST /media
6 Update Metadata PUT /media/123
7 Delete Media DELETE /media/123

Frontend (PyQt/PySide) GUI Features

The client application offers a seamless desktop experience for interacting with the backend API.

  • Data Visualization: Displaying media lists with key metadata.
  • CRUD Forms: Dedicated forms for creating, editing, and deleting records.
  • Interactive Search: Quick filtering and searching capabilities.
  • **Client-Server Communication:** Efficient handling of HTTP/JSON requests.

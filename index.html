Project Overview & Requirements

This project is a two-part software system‚Äîa Flask Backend and a PyQt/PySide Frontend‚Äîbuilt to manage metadata for an online library of books, films, and magazines. The Backend stores media information in JSON and provides RESTful APIs for listing, searching, filtering, creating, deleting, and viewing media entries. The Frontend uses these APIs to display the library, show detailed metadata, and handle user interactions. Users can add new media by clicking "Create New Media" and filling in the required details, or delete an entry by selecting it, clicking "Delete Selected Media," and confirming the action.

Backend (Flask)

The backend handles data persistence (JSON/Pickle) and HTTP API requests. It provides CRUD operations for all media types and exposes endpoints such as:

  • GET /media ‚Äî list all media
  • GET /media?category=Book ‚Äî filter by category
  • GET /media/{id} ‚Äî get metadata
  • POST /media ‚Äî create
  • PUT /media/{id} ‚Äî update
  • DELETE /media/{id} ‚Äî delete

Frontend (PyQt/PySide)

The frontend is a desktop GUI application that interacts with the backend API to display media lists, perform searches, and present metadata. It also offers create/edit/delete functionality via forms that call the API.

Backend (Flask) HTTP Endpoints

List Endpoints

List All Media: GET /media ‚Äî returns an array of media metadata objects.

List by Category: GET /media?category=<category> ‚Äî returns filtered results.

Display Metadata: GET /media/<id> ‚Äî returns the metadata for a single item.

Search & CRUD

Search by Name (Exact): GET /media?name=<name>

Create New Media: POST /media ‚Äî payload includes name, author, date, category.

Update Metadata: PUT /media/<id>

Delete Media: DELETE /media/<id>

Frontend (PyQt/PySide) GUI Features

CRUD Forms

Interactive search and category dropdown allow filtering results on the client side or via API queries.
